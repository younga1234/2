# 자동 개발 (병렬 에이전트 실행)

이 명령어는 필요한 모든 에이전트를 병렬로 자동 실행합니다.

## 1. 작업 분석 및 에이전트 할당

사용자 요청을 분석하고 다음 에이전트들을 병렬로 실행하세요:

### 백엔드 개발
```bash
/agent backend-architect "아키텍처 설계 및 API 구조 설계"
/agent api-developer "RESTful/GraphQL API 엔드포인트 구현"
/agent database-expert "데이터베이스 스키마 설계 및 최적화"
```

### 데이터 처리
```bash
/agent data-scientist "데이터 분석 및 시각화"
/agent ml-engineer "머신러닝 모델 개발 및 최적화"
```

### 성능 및 비동기
```bash
/agent async-specialist "비동기 처리 및 동시성 최적화"
/agent performance-optimizer "코드 성능 프로파일링 및 최적화"
```

### 품질 보증
```bash
/agent test-engineer "단위 테스트 및 통합 테스트 작성"
/agent code-reviewer "코드 품질 검토 및 리팩토링 제안"
/agent security-auditor "보안 취약점 감사 및 검증"
```

### DevOps 및 문서화
```bash
/agent devops-engineer "Docker, CI/CD 파이프라인 구축"
/agent documentation-writer "API 문서 및 README 작성"
```

## 2. 자동 실행 로직

다음 조건에 따라 에이전트를 자동으로 선택하세요:

### 조건 1: 프로젝트 타입 감지
```bash
# FastAPI/Django/Flask 프로젝트 감지
if [ -f "main.py" ] || [ -f "app.py" ] || [ -f "manage.py" ]; then
    echo "백엔드 프로젝트 감지됨"
    # backend-architect, api-developer, database-expert 실행
fi

# 데이터 분석 프로젝트 감지
if [ -f "*.ipynb" ] || grep -q "pandas\|numpy" requirements.txt 2>/dev/null; then
    echo "데이터 분석 프로젝트 감지됨"
    # data-scientist, ml-engineer 실행
fi

# 웹 스크래핑 프로젝트 감지
if grep -q "beautifulsoup4\|selenium\|scrapy" requirements.txt 2>/dev/null; then
    echo "웹 스크래핑 프로젝트 감지됨"
    # web-scraper 실행
fi
```

### 조건 2: 작업 타입에 따라
사용자 요청에 따라:

- **"API 개발"** → backend-architect, api-developer, test-engineer
- **"데이터 분석"** → data-scientist, ml-engineer
- **"성능 개선"** → performance-optimizer, async-specialist
- **"보안 검사"** → security-auditor, code-reviewer
- **"배포"** → devops-engineer, documentation-writer
- **"전체 검토"** → 모든 에이전트 병렬 실행

## 3. 병렬 실행 패턴

다음과 같이 에이전트를 그룹으로 묶어 병렬 실행하세요:

### 그룹 A: 설계 및 아키텍처 (동시 실행)
```bash
# 이 3개는 서로 독립적이므로 병렬 실행 가능
/agent backend-architect "백엔드 아키텍처 설계"
/agent database-expert "데이터베이스 스키마 설계"
/agent api-developer "API 설계 및 문서화"
```

### 그룹 B: 구현 및 최적화 (동시 실행)
```bash
# 설계가 완료된 후 병렬로 구현
/agent async-specialist "비동기 엔드포인트 구현"
/agent performance-optimizer "쿼리 최적화"
```

### 그룹 C: 품질 검증 (동시 실행)
```bash
# 구현이 완료된 후 병렬로 검증
/agent test-engineer "테스트 케이스 작성"
/agent code-reviewer "코드 리뷰"
/agent security-auditor "보안 감사"
```

### 그룹 D: 배포 준비 (동시 실행)
```bash
# 검증이 완료된 후 병렬로 배포 준비
/agent devops-engineer "Docker 및 CI/CD 설정"
/agent documentation-writer "최종 문서화"
```

## 4. 실행 예제

### 예제 1: 새 FastAPI 프로젝트 전체 개발
```bash
# 단계별 병렬 실행

# 1단계: 설계 (3개 에이전트 병렬)
/agent backend-architect "FastAPI 프로젝트 아키텍처 설계"
/agent database-expert "PostgreSQL 스키마 설계"
/agent api-developer "OpenAPI 스펙 작성"

# 2단계: 구현 (2개 에이전트 병렬)
/agent async-specialist "비동기 라우터 구현"
/agent database-expert "SQLAlchemy 모델 구현"

# 3단계: 품질 (3개 에이전트 병렬)
/agent test-engineer "pytest 테스트 작성"
/agent security-auditor "보안 검사"
/agent code-reviewer "코드 리뷰"

# 4단계: 배포 (2개 에이전트 병렬)
/agent devops-engineer "Docker 컨테이너화"
/agent documentation-writer "API 문서 완성"
```

### 예제 2: 데이터 분석 파이프라인
```bash
# 병렬 실행 (독립적 작업)
/agent data-scientist "EDA 및 데이터 정제"
/agent ml-engineer "예측 모델 개발"
/agent performance-optimizer "데이터 처리 최적화"
```

### 예제 3: 전체 코드베이스 검토
```bash
# 모든 검증 에이전트 동시 실행
/agent code-reviewer "전체 코드 리뷰"
/agent security-auditor "보안 취약점 스캔"
/agent performance-optimizer "성능 병목 분석"
/agent test-engineer "테스트 커버리지 확인"
```

## 5. 결과 통합

모든 에이전트가 완료되면 다음을 수행하세요:

### 결과 수집
- 각 에이전트의 출력을 `.claude/agent-reports/` 디렉토리에 저장
- 파일명: `{agent-name}_{timestamp}.md`

### 우선순위 분류
1. **Critical**: security-auditor에서 발견한 심각한 문제
2. **High**: code-reviewer, test-engineer에서 발견한 중요 문제
3. **Medium**: performance-optimizer에서 발견한 최적화 기회
4. **Low**: documentation-writer에서 제안한 문서 개선

### 종합 보고서 생성
```markdown
# 자동 개발 실행 보고서

## 실행 시간
- 시작: YYYY-MM-DD HH:MM:SS
- 종료: YYYY-MM-DD HH:MM:SS
- 소요 시간: X분

## 실행된 에이전트 (병렬)
- backend-architect ✅
- api-developer ✅
- database-expert ✅
- test-engineer ✅
- security-auditor ✅
- ...

## Critical 이슈 (즉시 수정 필요)
1. [security-auditor] SQL Injection 취약점 발견
2. [test-engineer] 커버리지 45% (목표 95%)

## High 우선순위
1. [code-reviewer] SOLID 원칙 위반 3건
2. [test-engineer] 통합 테스트 부족

## Medium 우선순위
1. [performance-optimizer] N+1 쿼리 문제
2. [async-specialist] 동기 코드를 비동기로 전환 권장

## Low 우선순위
1. [documentation-writer] README 업데이트 필요
2. [code-reviewer] 타입 힌트 추가 권장

## 다음 작업 제안
- [ ] Critical 이슈 즉시 수정
- [ ] High 우선순위 작업 계획
- [ ] 테스트 커버리지 95% 달성
- [ ] 문서 업데이트
```

## 6. 자동 실행 완료

최종 체크리스트:
- [ ] 모든 에이전트 실행 완료
- [ ] 결과 보고서 생성됨
- [ ] Critical 이슈 확인됨
- [ ] 다음 작업 계획 수립됨

위의 종합 보고서를 사용자에게 제공하세요.
